- **测试基础设施**：`conftest.py` 提供全局 `ToolRegistry` fixture；`pyproject.toml` 配置 `pytest-asyncio` 的 `auto` 模式以支持原生 `async` 测试函数。
- **Fake 驱动策略**：广泛使用 `FakeModel`（脚本化返回 `ModelResult`）和 `FakeMessage/ToolCall` 对象，实现无网络、无付费的离线确定性测试。
- **模块化覆盖**：测试文件按核心组件垂直切分，包括 `test_agent_loop.py`（主循环逻辑）、`test_session.py`（树状会话持久化与分支）、`test_extension.py`（子进程扩展生命周期）、`test_eval.py` 与 `test_dgm.py`（评估套件与候选者管理）。
- **隔离与环境**：所有文件系统操作均通过 `tmp_path` fixture 隔离在临时目录，确保测试不污染工作区；沙箱测试通过 `shutil.which` 动态跳过缺失的 Docker 环境。
