# M4.0 Eval + DGM-lite 基座 Plan

> 日期：2026-06-12  
> 范围：M4/v2 的第一块，只做 eval 护栏与 DGM-lite 候选归档基座。

## 1. 目标

把历史 `评测/` 脚本沉淀为库内可复用 eval 子系统，并建立最小 DGM-lite 闭环：

- eval 任务可由 Python API 和 `python -m mu.eval` 运行。
- 候选改进只在复制出来的 workspace 中叠加和验证。
- 通过 eval 的候选只进入 archive，不自动改主仓库。
- 离线单测可稳定验证；真实模型 eval 只作为环境变量齐全时的可选 smoke。

## 2. 范围内

- `EvalTask` / `EvalSuite` / `EvalResult` / `EvalRun` 小 API。
- 内置 `basic-coding` suite，复用原三个基础 coding-agent 任务。
- eval subprocess runner：每任务独立 workspace，外部 validator 独立验收。
- secret redaction：API key/token/secret 类环境变量不写入报告或 stdout/stderr 文件。
- `python -m mu.eval` 入口。
- DGM-lite 候选来源：
  - `--candidate-dir`：叠加允许路径下的文件。
  - `--patch`：应用只触碰允许路径的 patch。
  - `--generate`：在候选副本中让 μ 生成候选。
- 候选允许路径：
  - `.mu/extensions/*.py`
  - `.mu/prompts/*.{md,txt}`
  - `extensions/*.{py,md,txt}`
- archive：append-only `archive.jsonl` + `latest-summary.{json,md}`。

## 3. 范围外

- 不自动应用通过候选到主仓库。
- 不允许候选直接改 `mu/` core、session、工具执行、TUI、权限层。
- 不实现完整程序性记忆 / meta-tool 编译。
- 不实现投机 / 异步执行。
- 不把真实模型 API key 写入任何文件。

## 4. 验收

- 单测覆盖 eval 成功、失败、timeout、summary 输出和 secret redaction。
- 单测覆盖 DGM 候选隔离、archive best 标记、父子关系、disallowed path 拒绝。
- `readonly/workspace` 这类 restrictive permission 下拒绝扩展候选执行。
- 可选真实模型 smoke：当前 shell 设置 `MU_BASE_URL` / `MU_MODEL` / `MU_API_KEY` 后运行 `python -m mu.eval`，报告里不出现 key。
