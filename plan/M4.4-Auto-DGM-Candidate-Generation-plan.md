# M4.4 Auto DGM Candidate Generation 实施 Plan

> 阶段来源：`docs/Python复刻Pi-Roadmap-v1.md` M4 后续范围。M4.3 已具备 eval archive 与 guarded promotion，本阶段只把候选来源从手动提供推进到可控自动生成。

## 目标

自动生成一批 prompt-snippet DGM candidates，逐个隔离评测并写入 archive 排名；默认不应用主仓库，promotion 仍由 `mu.dgm_promote --apply` 显式授权。

## 范围

1. **Auto DGM API / CLI**
   - 新增 `mu.dgm_auto`。
   - `run_dgm_auto(...) -> DgmAutoRun` 串行生成 N 个候选，并对每个候选调用 `run_dgm_candidate(source_type="generate")`。
   - CLI：`python -m mu.dgm_auto --archive-dir dgm_archive --count 3 --description "..."`

2. **Prompt-only 自动候选**
   - 自动生成只允许 `.mu/prompts/*.md|*.txt`。
   - 生成提示明确禁止修改 core、tests、extensions、meta-tools、README、docs。
   - 生成 stdout/stderr 仍写入候选 `generation/` 目录并走 redaction。

3. **Archive / Summary**
   - 继续复用 `dgm_archive/archive.jsonl` 与 `latest-summary.*`。
   - 不改变 `DgmArchiveEntry` schema。
   - 新增 `dgm_archive/auto-runs/<run_id>/summary.json|md`，记录请求数量、生成数量、通过数量、best candidate、候选错误。

4. **Gate**
   - `python -m mu.eval_gate` 增加 `dgm_auto_fake_agent_smoke`。
   - smoke 覆盖自动生成 2 个 prompt candidates、逐个 archive、best 标记、secret scan。

## 验收

- `./.venv/bin/python -m pytest -q` 通过。
- `./.venv/bin/python -m mu.eval_gate --allow-missing-model` 离线 gate 通过，包含 DGM auto smoke。
- 设置真实模型环境变量后 full gate 通过，basic eval 保持 `3/3 PASS`。
- 手动 smoke：`python -m mu.dgm_auto --archive-dir dgm_archive --count 3 --description "..."` 生成 auto-run summary，随后可用 `python -m mu.dgm_promote --archive-dir dgm_archive` 查看/准备 best promotion。

## 非目标

- 不自动 apply / commit / stage / push。
- 不自动生成 extension 或 meta-tool。
- 不引入并行候选评测。
- 不改变 archive JSONL 格式、eval suite 默认行为、promotion 安全门禁。
