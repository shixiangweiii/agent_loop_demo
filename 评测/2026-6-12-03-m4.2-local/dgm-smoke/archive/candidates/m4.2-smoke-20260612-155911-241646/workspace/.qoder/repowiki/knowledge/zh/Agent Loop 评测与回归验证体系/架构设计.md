- **分层架构**：模块由多个按日期/版本组织的评测套件（如 `2026-6-11-01`）组成，每个套件包含独立的 Python 驱动脚本（`run_*.py`）和标准化的产物目录结构。
- **核心流程**：驱动脚本通过 `subprocess` 调用 `mu` CLI 或 `pytest`，在隔离的 `workspace` 中执行 Agent 任务，并通过内置 Validator（如运行测试用例）判定结果。
- **产物规范**：每次运行生成时间戳命名的 `runs/` 目录，内含各任务的 `agent_stdout.txt`、`validation.txt` 及汇总的 `summary.json/md`；顶层维护 `latest-summary.*` 指向最新结果。
- **扩展能力**：支持真实模型评估（`mu.eval`）、DGM-lite 候选者冒烟测试（通过 `fake_agent.py` 模拟）以及基于正则的敏感信息（Secret）扫描。
