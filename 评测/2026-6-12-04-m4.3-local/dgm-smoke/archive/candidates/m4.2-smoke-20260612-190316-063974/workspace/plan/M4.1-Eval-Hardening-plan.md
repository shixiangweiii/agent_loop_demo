# M4.1 Eval Hardening 实施 Plan

> 阶段来源：M4.0 回归评测暴露的 eval 护栏稳定性问题。范围只做 eval hardening，不进入程序性记忆、meta-tool 编译、投机执行或 DGM 自动应用。

## 目标

把 M4.0 的 eval 护栏从“可用”打磨到“可信”：原始 `python -m mu.eval` full run 应能直接稳定 `3/3 PASS`，不再依赖手动 revalidation 或 targeted rerun 兜底。

## 范围

1. **路径语义统一**
   - `project_root`、`run_root`、`run_dir`、`task_dir`、`workspace` 全部 `.resolve()`。
   - prompt、summary、`EvalResult` 中写入的路径全部为绝对路径。

2. **validator pytest 稳定化**
   - `run_pytest(workspace, test_files=...)` 显式传 `--rootdir <workspace>`。
   - 内置三个 basic task 分别只运行自己的测试文件，避免父级 `pyproject.toml testpaths=["tests"]` 漂移。

3. **库内 secret scan**
   - 新增 `scan_eval_artifacts_for_secrets(root, env=None, ignore_copied_fixtures=True)`。
   - 默认扫描 eval/DGM 过程产物与 summary；忽略复制出来的 `workspace` 源码 fixture。
   - 识别 `API_KEY`、`TOKEN`、`SECRET` 环境值与 `sk-...` 模式。

4. **CLI 输出与 summary 强化**
   - `python -m mu.eval` 打印绝对 run dir、`Passed: x/y`、secret scan 结果。
   - summary 明确区分 agent timeout、agent failure、validator failure、secret scan failure。

5. **Full Gate**
   - 新增 `python -m mu.eval_gate`。
   - 一键跑离线 pytest、真实模型 basic eval、DGM-lite fake-agent smoke。
   - 产物写入 `评测/<date-run>/` 或指定 `--run-root`，只读取运行时环境变量，不写 API key。

## 验收

- `./.venv/bin/python -m pytest -q` 通过。
- 设置真实模型环境变量后，`python -m mu.eval --run-root <绝对评测目录>/real-eval-runs --timeout 360` 原始 full run 直接 `3/3 PASS`。
- `python -m mu.eval_gate --run-root <绝对评测目录>` 产出：
  - `m4.1-eval-hardening-report.md`
  - `latest-summary.json`
  - `latest-summary.md`
  - `real-eval-runs/`
  - `dgm-smoke/archive/archive.jsonl`

## 非目标

- 不做程序性记忆。
- 不做 meta-tool 编译。
- 不做投机/异步执行。
- 不做 DGM 自动应用通过候选。
