# M4.3 Guarded DGM Promotion 实施 Plan

> 阶段来源：`docs/Python复刻Pi-Roadmap-v1.md` M4 后续范围。M4.0-M4.2 已具备 eval 护栏、候选 archive 和显式 meta-tool，本阶段把通过候选推进到可审计、可预演、显式授权后应用。

## 目标

把 DGM-lite archive 从“只归档候选”升级为“可生成 promotion patch，并在安全门禁后显式应用”：默认只写 patch/report，不改主仓库；`--apply` 才执行路径级脏检查、sandbox pytest preflight、备份和写回。

## 范围

1. **Promotion API / CLI**
   - 新增 `mu.dgm_promote`。
   - `prepare_dgm_promotion(...)` 从 archive 选择 `best` 或指定 candidate，生成 promotion patch 与 JSON/Markdown summary。
   - `apply_dgm_promotion(...)` 只应用已准备好的 promotion。
   - CLI：`python -m mu.dgm_promote --archive-dir dgm_archive [--candidate best|id] [--apply]`。

2. **Promotion 门禁**
   - 只允许 `passed == total` 且 `total > 0` 的 archive candidate。
   - 要求候选 eval summary 存在且 secret scan PASS。
   - promotion 产物继续 secret scan；命中真实 env secret 或 `sk-...` 模式时拒绝 apply。
   - 继续复用 DGM-lite allowed path：`.mu/prompts`、`.mu/extensions`、`.mu/metatools`、`extensions`。

3. **显式 apply**
   - 非 git repo 允许 prepare，拒绝 `--apply`。
   - `--apply` 只检查目标 `changed_paths` 的 git dirty 状态，无关脏文件不阻塞。
   - apply 前复制当前项目到 promotion sandbox，叠加候选变更，运行 `pytest -q`。
   - 写回前备份目标路径；失败时尝试恢复。
   - 不自动 commit / stage / push。

4. **Gate**
   - `python -m mu.eval_gate` 增加 `dgm_promotion_smoke`。
   - smoke 覆盖 archive → promotion patch → dirty-path 拒绝 → sandbox pytest → apply 写回。

## 验收

- `./.venv/bin/python -m pytest -q` 通过。
- `python -m mu.eval_gate --allow-missing-model` 离线 gate 通过，包含 DGM promotion smoke。
- 设置真实模型环境变量后 full gate 通过，basic eval 保持 `3/3 PASS`。
- 手动 smoke：先用 `python -m mu.dgm` 产生通过候选，再用 `python -m mu.dgm_promote` 生成 patch，最后显式 `--apply`。

## 非目标

- 不自动生成新候选。
- 不自动学习 / 自动生成 prompt 或 meta-tool。
- 不自动 commit、branch、PR。
- 不改变 archive JSONL 格式、eval suite 默认行为、extension/meta-tool 加载语义。
