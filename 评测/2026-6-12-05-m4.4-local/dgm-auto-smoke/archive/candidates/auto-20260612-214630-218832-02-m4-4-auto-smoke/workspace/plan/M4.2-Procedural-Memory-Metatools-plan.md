# M4.2 Procedural Memory / Meta-tools 实施 Plan

> 阶段来源：`docs/Python复刻Pi-Roadmap-v1.md` M4 后续范围。M4.1 已把 eval 护栏硬化，本阶段开始把高频工作流固化为可复用 meta-tool。

## 目标

把 M3.5 的 `code-action` 从“单次临场组合工具”升级为“可复用的程序性记忆”：把已验证、高频工具调用流程写成 repo-local JSON spec，显式加载后注册为普通 tool，供模型直接调用。

## 范围

1. **Repo-local meta-tool**
   - 从 `.mu/metatools/*.json` 或 `MU_METATOOL_DIR` 加载。
   - 每个 spec 注册为普通 tool，走 `ToolRegistry.execute`、事件流、权限策略与 OpenAI tool schema。
   - 执行环境提供 `args` 与 `mu.read/write/edit/bash/call/log/result`。

2. **Spec v1**
   - JSON 字段固定为 `name`、`version`、`description`、`parameters`、`code`。
   - `name` 必须匹配 Python/OpenAI function name 风格。
   - `parameters` 必须是 JSON Schema object。
   - `code` 必须是内联字符串；不支持外部 Python 文件引用。

3. **显式启用与安全边界**
   - CLI 新增 `--metatools` / `--metatool-dir`，环境变量为 `MU_METATOOLS=1` / `MU_METATOOL_DIR`。
   - 默认关闭；开启后注册 `list_metatools` 与 `reload_metatools`。
   - 所有 meta-tool 固定 `code_exec` capability；`readonly/workspace` 会整体拦截。

4. **DGM / eval gate**
   - DGM candidate allowlist 增加 `.mu/metatools/*.json`。
   - DGM eval 环境显式设置 `MU_METATOOL_DIR` 指向候选 workspace。
   - Full gate 增加 meta-tool fake-model smoke。

## 验收

- `./.venv/bin/python -m pytest -q` 通过。
- `python -m mu.eval_gate --allow-missing-model` 离线 gate 通过，包含 meta-tool smoke。
- 设置真实模型环境变量后 full gate 通过，basic eval 保持 `3/3 PASS`。
- 手动创建 `.mu/metatools/quick_pytest.json` 后，`mu --metatools` 可直接看到并调用该 meta-tool。

## 非目标

- 不做自动学习 / 自动生成 / 自动应用。
- 不做 prompt 记忆。
- 不做投机/异步执行。
- 不做 meta-tool 市场/跨 repo 分发。
